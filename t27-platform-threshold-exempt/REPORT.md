# t27 REPORT — 模式 10a 补「平台/交付规范硬约束阈值免原理引用」

**日期**：2026-06-21
**判定**：✅ **KEEP**（B 35 vs A 32，dominant baseline，入配）
**来源**：t25 对照 joeseesun/qiaomu-icon-generator 识别的派生微调 Gap 3（favicon 32px 是平台事实非品味）。

## 假设

模式 10a 规则 2「每条阈值必须带依据，裸数字=堆料」对**平台硬约束阈值**误伤——favicon 16/32px、邮件 600px 是查得到的平台事实，注明出处即够，硬挂 Tufte/WCAG 式原理反而荒谬。补丁加一条例外：平台/交付规范硬约束阈值注明出处即免原理引用，但「别把品味数字伪装成平台约束逃引用，分不清从严」。

## 方法（t24/t25 严测）

单变量（arm B 框架仅多 10a 例外条）、对称元指令、4 场景（含危险点 + noise）、盲评带 noise penalty。打乱映射：S1/S3 甲=B、S2/S4 甲=A。

## 为什么 KEEP（补丁效果直接显形）

1. **clear-hit S1 是决胜证据**：A 遵循 10a 现行死规则「每条阈值必须带依据」，被逼给 favicon 16/32px 这个**纯平台事实**硬挂三个学术引用。盲评员（不知哪个加了例外）独立判定这是「把平台事实包装成研究结论，别扭且可疑」，给 A 仅 6.5。B 有了例外，直接「注明是 favicon 显示尺寸」，判「正中要害」8.5。差 2.0 全部来自补丁修掉的过度套用。
2. **危险点 S2 通过**：补丁最大风险是「例外变漏洞，品味数字伪装成平台约束逃引用」。实测 B 没这么干——留白/主色/字重照样带 Calimari/Cleveland&McGill/WCAG 依据，与 A 打平。「分不清从严」的兜底起了作用。
3. **gray S3**：B 把平台约束（600px 注出处）与品味（行高/配色带依据）做成两张分列表，比 A 散落小节更清晰。
4. **noise S4**：纯转换任务，两 arm 都没塞阈值机制，补丁无副作用。

补丁修的是真缺口（10a 一刀切误伤平台事实），且危险点不破防 → dominant baseline，KEEP。
