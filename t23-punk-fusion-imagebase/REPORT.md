# t23 — image-base「具名风格融合」节 AB 测试

**日期**：2026-06-20
**起因**：@AdrianPunk115 开源 Punk-Skill（github.com/adrianpunk/Punk-Skill），把视觉风格做成可调用 skill。决定不全量安装，只把 provider 无关的两点纪律（融合而非拼接 + 封面标题三层级）吸收进 `~/.claude/references/image-base.md`。按「外部整合必须 AB，无提升回滚」跑验证。

## 设计

- **唯一变量**：image-base.md 新增的「套用具名风格时：融合，不是拼接」一节（3 条 bullet：融合而非拼接 / 不可见不写 / 封面标题三层级）。两版其余完全相同（同样的优先级自检 + 6-block + 跨类目避坑 + 同一份具名风格描述）。
- **A（baseline）= ctx_A.md**：无融合节。
- **B（改后）= ctx_B.md**：= A + 融合节。
- **3 场景**：S1 麦肯锡/咨询风（X 5:2，结构简单）、S2 巨型透视字（小红书 3:4，叙事丰富）、S3 极简黑白编辑风（公众号 2.35:1，长标题，测三层级）。
- **生成**：6 个 Sonnet agent（3 场景 × 2 版本），各产一条最终封面生图 prompt，互不知对方存在、不知是 AB 测试。
- **评审**：3 个 Sonnet 盲评 agent（每场景一个），prompt 顺序随机打乱、不标 A/B、用中立的封面质量五项标准（一眼可辨主题 / 风格独特性非拼贴 / 标题完整可读+层级 / 可执行具体度 / 整体刻意设计感），避免循环论证。

## 结果

| 场景 | 改前 A | 改后 B | 胜者 | margin |
|------|------:|------:|------|------:|
| S1 麦肯锡风 | 41 | 42 | B | +1 |
| S2 巨型透视字 | 37 | 44 | B | +7 |
| S3 长标题黑白编辑 | 29 | 44 | B | +15 |
| **合计** | **107** | **130** | **B** | **+23** |

B 三场全胜，dominant baseline。

## 差距来源（盲评 agent 原话归纳）

- **S3（+15，最大）**：A 把长标题截成「复盘 / 的三个陷阱」，丢掉「无效」「反思」「认知科学」核心词——不是精简是截断；B 用 A/B/C 三层（无效大字锚点 + 完整标题 + 副标题）把长标题设计消化掉了。三层级纪律直接拉开差距。同时 B 的「字即裂缝、笔画与裂缝走向对齐」让文字与图像共生而非叠加，正是该风格精髓；A 停留在「左墙+右文字区」的通用分区。
- **S2（+7）**：B 给出完整叙事空间（工业隧道 + 过期项目进度表 + 手撕日历「30」+ 逆光人影），每个元素服务「废弃两年、30 天救活」；A 场景几乎只有字本身，人影和「30天」是贴上去的标签。融合纪律推动了「用风格语言实现每一块」。
- **S1（+1，最小）**：结构本就简单（两区对比），融合节的边际价值低；B 略胜在「卖结果」单一核心论断更符合麦肯锡封面特征 + 给了具体色值降低出图漂移。

## 结论

**Keep。** `~/.claude/references/image-base.md` 的「套用具名风格时：融合，不是拼接」节保留。

价值分布有梯度，诚实记下来：**融合纪律的收益集中在长标题文 / 叙事丰富文**（S3/S2），对结构简单的对比型封面（S1）边际收益小。三层级 bullet 是 S3 拉开 15 分的主因——长标题封面是这条规则的核心适用场景。

## 不吸收项（与本测试无关，备查）

Punk-Skill 的「确认门」（每次出图前停下来等用户确认）与 Edon 的 no-interrupt 执行偏好冲突；「风格库 + metadata 门控 + 渐进披露」属于全量安装那套架构，已决定不装。本测试只验证了 provider 无关的两点纪律。
