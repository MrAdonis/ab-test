# 封面诊断 / 自查（B 臂独有，蒸馏自 ponyo-cover-anchor-system）

封面成败不在"好不好看",在用户 3 秒扫信息流时能不能被停住。决定点击的只有两个变量:
- **信息密度** — 利益点在缩略图尺寸下清不清楚
- **视觉锚点** — 有没有一个位置让眼睛停住(人脸/大号数字/截图/前后对比/强轮廓物体);若所有元素等权,即无锚点

## 诊断既有封面（用户给图问"为什么不行/帮我改"时）

输出固定结构:
```
## 诊断
模板契合: <冲突/数字/截图/情绪/混乱>
主问题: <一句话>
点击风险: <为什么在 feed 里会被划走>

## 打分（各 1-10）
信息密度 / 视觉锚点 / 缩略图可读性 / 对比度 / 可信度

## 修法（按影响排序）
1. <影响最大的改动>  2. <次>  3. <三>

## 替换方案
<可直接用的成品封面 prompt>
```
诊断要直接,不奉承弱封面。

## 可检验硬门（出方案/自查都过一遍）

- **80px 缩略图测试**:封面缩到约 80px 宽,主标题仍可读 + 一个锚点仍明显 + 色块仍分明 = 合格。变糊/细节消失/低对比浅字看不见 = 不合格。
- **对比度**:主标题 vs 背景 ≥4.5:1;缩略图模式建议 ≥7:1。安全对:黑/黄、白/红、白/深蓝、黑/珊瑚。危险对:浅绿/白、米色/浅灰、红/橙。
- **标题字数**:冲突 6-12 字 / 数字型 数字+4-10 字 / 截图 8-14 字 / 情绪 ≤8 字;超 16 字必拆(封面短标题做钩子,完整标题留正文)。
- **文字层级**:主标题占文字权重 60-75% / 辅助行 20-30% / 角标 5-10%。角标比主标题还抢眼就压掉。
- **锚点位置**:Z/F 扫描决定最强位是左上和上方水平中心。默认——标题上 1/3,数字/对比/手机居中,人脸/产品在下中部,品类角标左上;最重要锚点别丢右下角。

## 改版顺序（既有封面要重做时）
先改标题 → 选一个模板(非混搭)→ 选一个锚点 → 定两主色一撞色 → 定标题位置 → 出成品 prompt → 跑 80px 缩略图测试。

## 标题改写模板（标题太长/太抽象时）
`不要再 <错误行为>` / `<数字> 个 <结果> 方法` / `改前 VS 改后` / `<工具> 这样用` / `先别 <常见错误>` / `<情绪状态>,然后呢`
