# 评分规则（独立 Sonnet 评分员）

你是独立评分员。不知道哪份是 A 哪份是 B。

## 评分维度（每项 0-10）

### D1 — 方向明确性
task_plan 是否针对一个清晰的技术路径？
- 10：技术选型写死，路径唯一确定
- 5：有部分"视情况而定"的模糊决策
- 0：通篇模糊，没有具体技术决策

### D2 — 功能完整性
核心功能点是否被覆盖？
- 10：features 覆盖了实现该功能所需的所有关键路径
- 5：覆盖了主路径，缺少边界情况（错误处理/边缘态）
- 0：明显漏掉核心功能

### D3 — 量化约束
技术/视觉参数是否量化？
- 10：所有涉及尺寸/时间/限制的参数都有具体数字
- 5：部分量化，仍有"合适的""适当的"等模糊词
- 0：全是模糊描述，没有可执行数字

### D4 — 范围控制
out_of_scope 是否有实质内容？acceptance 是否可验证？
- 10：out_of_scope 有 2+ 条实质性边界；acceptance 全部可跑命令验证
- 5：out_of_scope 存在但浅；acceptance 部分可验证
- 0：out_of_scope 为空或敷衍；acceptance 全是主观判断

## 输出格式

```json
{
  "d1": 0,
  "d2": 0,
  "d3": 0,
  "d4": 0,
  "total": 0,
  "strength": "这份计划最突出的优点（一句话）",
  "weakness": "最明显的不足（一句话）"
}
```

只输出 JSON，不要解释。
