# S3 安全审查 — Arm B

## 1. 清单逐条对照

1. **无密钥/token 在暂存区** — 不命中。代码无任何密钥、token、凭证。
2. **无新增 OWASP Top 10 漏洞** — 不命中。无 Web 交互层、无数据库、无外部 I/O，不引入 Top 10 漏洞。
3. **appsec 面对照** — 跳过。输入来自调用方直接传参（spot/strike/premium 均为数值型），无外部来源（无表单/URL/上传/反序列化/SSRF）；输出为纯 Python list，无下游 sink（无 HTML 渲染、无 SQL、无 shell、无模板引擎）。满足"输入无外部来源 且 输出无下游 sink"的跳过条件。
4. **fail-open 默认检测** — 不命中。代码无环境变量读取、无配置加载、无认证开关、无 fallback 默认值。全部逻辑为纯数值计算，无安全语义的默认值路径。

## 2. 审查结论

**无安全问题。**

这是纯计算函数：期权 payoff 数学公式、浮点范围生成器、曲线数据构建。没有 I/O、没有外部输入、没有凭证、没有认证逻辑，四条清单项全部不触发或满足跳过条件。

唯一值得记录的非安全注意事项：`frange` 用浮点累加（`x += step`），累积误差会随区间变大漂移，适合绘图但不适合需要精确边界的场景——这是数值稳定性问题，非安全问题。

## 3. 自述

应用了清单第 1、2、4 条的主动扫描：确认无密钥/token，确认无 Web 框架/DB/shell 调用，确认无环境变量读取或 fallback 默认值。

第 3 条（appsec 面对照）按规则走了"跳过判定"：逐一检查五个触发面——①不可信输入：函数参数由调用方传入，无表单/URL/上传/反序列化/SSRF；②鉴权：无；③输出注入：输出为 Python list，无 HTML/SQL/shell/模板 sink；④加密/口令：无；⑤CSRF/CORS：无 HTTP 层。五面全部不触发，依规跳过，未套用 OWASP Cheat Sheet（对纯计算套用是 noise）。
