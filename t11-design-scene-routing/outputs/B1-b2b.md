## Design Read

**场景识别**：多店铺 Etsy 订单管理后台，核心诉求是高密度信息浏览 + 批量操作效率，非展示型界面。

> Design Read：Etsy ERP 多店铺订单管理后台（数据表格），VARIANCE **2**, MOTION **1**, DENSITY **9**

命中信号关键词：ERP / 管理 / 后台 / 表格 → 走 **§B2B工具规范**。

---

## 规则节选择理由

B2B 工具规范的核心价值观是"密度优先 + 操作响应快"。订单管理后台的用户（运营/仓管）每天要处理几百到几千条订单，他们需要的是：在不滚动的情况下看到尽量多的行，快速扫描状态，批量勾选。任何视觉装饰（毛玻璃/大圆角/bounce）都是在消耗他们的注意力和等待时间，反而是伤害。

## 关键参数

| 参数 | 值 | 备注 |
|------|-----|------|
| 正文字号 | **13px** | 订单号、买家名、商品名等主字段 |
| 副字段字号 | **12px** | 日期、SKU、备注、平台标签 |
| 区块标题 | **16px** | 页面标题、分组 header |
| 行高 | **1.4** | 紧凑行，不是 web 默认 1.6 |
| 数字列 | `font-variant-numeric: tabular-nums` | 金额、数量对齐 |
| 字体 | Inter / system-ui，禁 serif | |
| 圆角 | **4px**（行内标签）/ **6px**（按钮）/ **8px**（卡片/弹窗） | Dialog 例外可到 12px |
| 动画时长上限 | **150ms ease-out**（hover/focus）| 操作响应感知阈值，超过即显迟钝 |
| 阴影 | **仅 hover 时显示**，`box-shadow: 0 1px 4px rgba(0,0,0,0.12)`，静止状态无阴影 | |
| 网格基准 | **8px** | 行高 = 40px，间距 8/16/24px |

## 四个必备组件状态

### 1. Loading
- 表格主体替换为**行级 skeleton**：保留列宽结构，每行用灰色矩形（高度同数据行 40px）占位，明度交替（`#F3F4F6` / `#E5E7EB`）
- skeleton 条带有 shimmer 扫光动画，`1.2s linear infinite`，不超过 150ms 单帧感知阈值
- 顶部筛选栏、列头正常显示，表达"结构就绪，数据在来"
- 批量操作按钮 disabled 状态，避免空操作

### 2. Empty
- 表格区域居中：64×64 灰色图标（空纸箱或列表图标）
- 图标下方 **14px** 主说明："没有找到订单"
- 次说明 **12px** `#6B7280`：根据当前筛选状态动态给出："当前筛选条件下无结果，试试调整日期范围或店铺筛选"
- 下方 CTA 按钮（outlined 样式）："清除筛选" 或 "查看所有订单"
- **必须有 next action**，不能死页面

### 3. Error
- **行级错误**（单行数据拉取失败）：该行背景 `#FEF2F2`，最右侧显示 `重试` 文字链，12px 红色说明
- **全局错误**（接口超时/500）：右下角 toast，红色背景，文案格式严格遵守：**"陈述事实 + 给行动"**
  - ✅ "获取订单列表失败（网络超时），点击重试"
  - ❌ "哎呀，出了点小问题～" （禁卖萌）
- toast 持续 **4s 自动消失**，消失前提供手动关闭 ×
- 表格保留上一次成功加载的数据，顶部显示 warning bar："数据可能未更新"

### 4. Success
- 批量操作（标记发货、导出）完成后：右下角 toast，绿色背景，1.5s 自动消失
- 文案样例："已标记 12 个订单为已发货"（含具体数量）
- **无 confetti，无 modal 确认弹窗**
- 新订单异步推送到列表时，新增行有 `0.15s` 背景高亮 fade（`#ECFDF5` → 透明），不跳动不 bounce

## 禁忌（不该做）

**1. 毛玻璃和暗色装饰卡片**
表格上方不加 `backdrop-filter: blur()` 的浮层工具栏，不用深色卡片包裹订单行。这些效果在 GPU 合成层上渲染慢，后台工具用户一天要看几千次，视觉噪声是真实的认知成本。

**2. 正文字号 ≥16px**
表格正文用 16px 意味着同屏显示的行数减少约 25%。运营人员需要在不滚动的情况下看到更多行来做决策。16px 是博客字号，不是后台字号。

**3. 弹簧/bounce 动画**
列表行展开、侧边栏滑入、toast 出现，一律禁用 `cubic-bezier` bounce 或 `spring` 弹性物理。操作工具的用户已经知道点击会有反应，不需要动效"确认"——他们需要的是结果出现得快，不是出现得有趣。动画风格定在 `ease-out`，时长封顶 150ms。

**4. hover 常驻阴影**（附加）
行 hover 状态可以有轻微阴影或背景色变化，但静止状态的每一行都有 `box-shadow` 会让表格看起来像一堆浮动卡片，破坏扫描连续性。阴影只在 hover 时出现，其余时间平面。
