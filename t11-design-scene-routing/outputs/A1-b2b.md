**Design Read：Etsy ERP 多店铺订单管理后台，数据面板场景，VARIANCE 2, MOTION 2, DENSITY 9**

---

## 选用规则节

**数据面板**（VARIANCE 2-3 / MOTION 2-3 / DENSITY 8-10）

理由：订单列表是操作型界面，用户目标是扫描、过滤、处理订单，不是感受品牌。高密度让每屏装更多行，低动效减少认知干扰，严格网格保证多列对齐可读。品牌落地页的大字 hero 和作品集的破格构图在这里是反模式。

---

## 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 表头字号 | 11px / weight 600 | 全大写或加字间距，与数据行区分 |
| 数据行字号 | 13px / weight 400 | 保证单行高密度可读 |
| 行高 | 40px（紧凑）/ 48px（宽松可选） | 固定行高，避免多行内容撑开破对齐 |
| 圆角 | 整表 0px，badge/tag 4px，按钮 6px | 表格本身用直角，只在内嵌元素用小圆角 |
| 动画时长上限 | 150ms | 排序切换 / 行 hover 背景，不超 150ms；超过即感觉卡 |
| 网格基准 | 8px | 列内 padding 16px，行间距 1px border |

## 四个必备组件状态

**Loading**
骨架屏，不用 spinner。每行用灰色矩形块（`#E5E7EB`）替代真实数据，宽度按列比例模拟（订单号列 80px，买家列 120px，金额列 60px）。pulse 动画 1.5s ease-in-out 循环，亮暗区间 `opacity 0.4 → 1`。行数固定为上一次加载行数，或默认 10 行，避免骨架屏高度跳变。

**Empty**
区分两种空态：①「没有任何订单」：插图（线框风箱子）+ 一句话描述 + 主 CTA（去店铺同步）；②「过滤后无结果」：不用插图，只写「没有匹配"已发货 + 过去7天"的订单」+ 「清除筛选」文字链。两种不能用同一模板，混淆会误导用户以为店铺真的没单。

**Error**
行内 banner，不全屏遮罩。表头下方粘一条红色 `#FEF2F2` 横条，左侧圆圈感叹号图标，中间写具体错误（「Etsy API 请求超时，请求ID: xxx」），右侧「重试」按钮。保留上一次成功加载的数据行但置灰 `opacity 0.5`，让用户看到数据轮廓，知道不是空数据问题。

**Success**
即表格正常态。表头固定吸顶（`position: sticky; top: 0`），每行 hover 背景 `#F9FAFB`，选中行 `#EFF6FF`。多店铺用左侧彩色竖线区分（店铺 A = `#3B82F6`，店铺 B = `#10B981`），不用全行背色避免列信息被颜色淹没。

---

## 该场景的 3 条禁忌

**1. 禁止浮动操作按钮覆盖数据列**
行内操作（发货、备注、标记）不用悬浮卡片弹出覆盖相邻行。改用行尾固定宽操作列（64px），或单独操作抽屉从右侧滑入。鼠标移动过程中误触遮罩是数据表格最高频的操作失误。

**2. 禁止列宽随内容自动伸缩**
固定每列最大宽度并 `overflow: hidden; text-overflow: ellipsis`。买家名称、商品标题长短不一，自动伸缩会让整行在不同数据间跳宽，用户无法建立列位置肌肉记忆，扫描效率下降 30%+。

**3. 禁止用颜色作为订单状态的唯一编码**
「待发货=红、处理中=黄、已完成=绿」不够，必须加文字 label（`未付款` `已发货` 等）。理由双重：色觉差异用户（约 8% 男性）无法区分红绿；跨店铺颜色语义可能冲突（某些 Etsy seller 用红色标记 VIP 订单而非异常订单）。badge = 颜色 + 文字，缺一不可。
