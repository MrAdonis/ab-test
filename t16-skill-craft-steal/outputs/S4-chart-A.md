## 核心规则

### 触发条件

满足以下任一条，启动本 skill：
- 用户提供一组数值数据，并要求对比、趋势、分布或排名
- 用户说"画图"/"出图"/"可视化"/"对比一下"，且上下文有多组数值
- 任务输出是给他人看的报告、总结、演示文稿，需要附图

### 图表类型选择（硬规则）

| 数据结构 | 图表类型 |
|----------|---------|
| 分类对比（≤10 组） | 横向条形图（bar horizontal） |
| 时间趋势 | 折线图（line） |
| 占比（≤6 类） | 饼图或圆环图 |
| 双变量关系 | 散点图 |
| 多指标横向对比 | 分组条形图；**禁用**雷达图（难读精确值） |

**禁止**：3D 图、渐变填充、超过 3 种以上配色、阴影效果、双 Y 轴（除非用户明确要求）。

### 视觉参数（低自由度，硬执行）

- 字体：标题 16px / 轴标签 12px / 数据标注 11px，全部无衬线
- 配色：主色 1-2 个，对比色仅用于需要突出的数据点；使用对色盲友好的配色方案（蓝橙、蓝红而非红绿）
- 图例：≤3 类时去掉图例，直接标注在数据旁；>3 类图例置顶左对齐
- 网格线：水平浅灰虚线，不画垂直网格线
- 坐标轴：Y 轴从 0 开始，除非数据全在高区间（差异 < 10% 量程），此时显示截断标志
- 数据标注：条形图每条标数值；折线图只标端点和极值

### "清晰、一眼能看懂、专业"的可执行解读

| 主观目标 | 可验证标准 |
|----------|-----------|
| 清晰 | 最长的数据标签不超出图边界；主标题 ≤ 20 字，说明"什么和什么比" |
| 一眼能看懂 | 最重要的结论在标题或副标题直接写出（如"Q2 增速最快"），不让读者自己归纳 |
| 专业不花哨 | 无动画、无图标装饰、无背景图片；空白大于数据墨水时删元素，不加元素 |

### 输出格式

生成 Python（matplotlib / plotly）脚本，输出 PNG，分辨率 150 dpi，宽度 900px。不内联输出 base64，给文件路径。
