**日期**：2026-06-11
**来源**：`github.com/xiaohuailabs/xiaohu-video-translate`（X @xiaohu，"学习一下"）
**测什么**：从该项目 SKILL.md 提炼的 6 个 skill 封装手法中，用户点名 #2、#4 两条，验证嫁接进 `~/.claude/references/skill-design-patterns.md` 是否真有增益（B>A 才采纳，否则回滚 + 记 rejected buffer）。

## 两条候选手法

- **#2 description 双重防御**：把"触发词覆盖 + 防半途执行（不能 build 完/审核完就报成功）"都钉死在 description 字段里，不留 agent 自由判断空间。
- **#4 主观品味→可测规则（带依据）**：把"好不好看/专业不专业"这类主观品味尽量拆成 agent 可确定性执行的阈值（留白比例、颜色数、字号），每条阈值带依据/原理，真无法量化的部分诚实保留文字。

## 实验设计

- 2 track × 2 scenario = 4 场景，每场景 A（基线指导）/ B（注入候选规则）各生成一份。
- 生成：4 个隔离 `general-purpose` 子代理（self-contained prompt，不继承主会话 xiaohu 分析，防污染基线）。
- 评分：4 个 Sonnet 盲评裁判，每场景甲/乙匿名 + 位置每场景翻转（防位置偏置），5 维各 10 分（满分 50）。

| 场景 | Track | 任务 |
|------|-------|------|
| S1 | 1 (#2) | 部署 skill 的 description |
| S2 | 1 (#2) | 美股小报 skill 的 description |
| S3 | 2 (#4) | 文章封面 skill 的核心规则 |
| S4 | 2 (#4) | 数据图表 skill 的核心规则 |

## 裁判关键 reasoning

- **S2（#2 失败的关键场景）**：基线自发写出更厚的防半途内容——点名 3 个已知失败模式（生成完停住/审核完不发布/命令返回 0 但没上线）+ 审核三态评级（✅/⚠️/❌）各带后续行为。注入版 B 反而更薄。裁判："乙（基线）每段都要求可验证完成信号，比甲的'管线未跑完不算 done'这句泛化断言执行力强得多。"
- **S3/S4（#4 成功的关键维度）**：两场都把"依据扎实"判为决胜维度。B 的阈值每条挂依据（Calimari Rule / WCAG AAA 7:1 / Cleveland & McGill 1984 / Tufte 数据墨水比），A 的阈值是"光溜溜的数字，没有任何来源"。S4 裁判："乙（A 基线）的禁超 3 色/字号 16/12/11 是光溜溜的数字，在'依据扎实'维度输得彻底。"

## 诊断：#2 为什么失败

不是 #2 这条手法本身错，是**我的配置已有等效覆盖，再加即堆料**（违反 bounded-edit「只增不删=堆料」）：

- `skill-design-patterns.md` 模式 9：「description（L1）是唯一触发机制」——触发层已写死。
- `skill-design-patterns.md` 模式 6：「COMPLETED ≠ 成功」陷阱点名——防半途已写死。
- `rules/coding-dod.md` 真实信号验收闸：三态分流（2xx 完成 / 401 前置未满足 / 5xx 真 bug）——防半途的可执行版已写死。

AB 实测里，基线指导（已含模式 6 的"COMPLETED≠成功"）自发产出的防半途内容比注入"双重防御"更厚。证明这条增量在我的配置里没有边际价值。

## 诊断：#4 为什么成功

#4 命中我配置的真空地带。现有模式 10（Freedom Matching）按**技术脆弱度**分配自由度——"封面好不好看"会被归类为低脆弱→高自由→纯文字描述。#4 修正这个盲区：主观品味也该尽量拆成带依据的可测阈值，只对真无法量化的残余保留文字。两场 AB 的决胜维度（依据扎实）正是注入规则里特意保留的"规则要带依据否则堆料"那条纪律（源自 `feedback_design_system_distill_rejected.md` 的教训）。

## 决策

- ✅ **#4 采纳**：作为模式 10 的子节 10a「主观品味→可测规则（带依据）」并入 `skill-design-patterns.md`，加一条自检。是模式 10 缺的那一格（脆弱度轴 vs 品味轴），不是重复。
- ❌ **#2 否决**：记入 rejected buffer。配置已有模式 6/9 + coding-dod 真实信号闸三处覆盖，加它纯堆料。

> **bounded-edit 纪律体现**：本轮原打算加 2 条手法，AB 实测杀掉 1 条，净采纳 1 条——正是「整轮优化只接受 1-4 次编辑，只增不删=appending」的实操。被否的 #2 即本轮的"删除"。
