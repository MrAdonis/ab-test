# t28 REPORT — 吸收 renwei-writing「改稿场景减法纪律」进 writing.md

**日期**：2026-06-21
**判定**：❌ **REJECT**（A 32 vs B 27.5，baseline 四场全胜，不入配）
**来源**：X @king1818888 推的开源 skill `orange2ai/renwei-writing`（人味儿写作）。对照 `rules/writing.md` 识别出唯一真盲区——writing.md 一直假设"从零写"，没区分"打磨已有初稿"，且「具体坐标系/人物画像法」在改稿场景会反噬（替作者编造记忆）。

## 假设

给 writing.md 加一节「改稿场景的减法纪律」：润色他人/自己初稿时反转原创纪律——不主动加时间场景锚点、语气词当手迹不删、抽象词换具体词要克制、打磨N遍≠逐级加工、上限是隐形。预期在"改富含手迹的初稿"场景，补丁版（B）比 baseline（A）更能保住人味。

## 方法

单变量（两臂同读 shared-rules.md 写作准则节选 + 全局 writing.md，arm B 仅多读 patch.md 这一节）。4 场景：S1 clear-hit（润色带语气词/白描的卖房随笔）、S2 危险点（原创推文，测补丁是否误伤具体细节）、S3 gray（改半 AI 味产品介绍，含该删的拔高 + 真含糊处 + 作者口吻）、S4 noise（精简成标题）。打乱映射后盲标签 甲/乙 交独立 Sonnet 评分员，评分员不知哪臂带补丁。映射：S1 甲=B/乙=A，S2 甲=A/乙=B，S3 甲=B/乙=A，S4 甲=A/乙=B。

## 为什么 REJECT

1. **baseline 已自然覆盖**：writing.md 的「编辑不是代笔/保住味道」+ 任务自身"别改没了我的味道"的框定，已让 baseline 在 S1 自然克制——保住"哦,这事终于结束了"、语气词、白描。显式补丁是冗余。
2. **补丁把克制推过头成机械保守**：S3 是最硬的反证——补丁的"不主动加、保留作者措辞"纪律让 B 不敢重塑，结果**保住了该删的企业腔、删掉了该留的口语**，比 baseline 更没人味。减法纪律写成显式清单后，模型机械执行"少动"，反而失去了 baseline 那种"看整体该怎样就怎样"的判断。
3. **危险点未完全通过**：S2 原创场景 B 也轻微落后，说明补丁对"何时该加细节"的判断有干扰外溢。

## 配置缺口已被覆盖（无需补）

复盘"真盲区"假设本身：writing.md 确实没有显式的改稿小节，但 baseline 行为证明这个判断**已隐含在现有规则 + 任务上下文里**，不需要写成独立条款。这与历史 reject 同型——`feedback_design_system_distill_rejected`（prose 加规则两次无提升，饱和）、`feedback_description_double_defense_rejected`（已有模式多处覆盖，加即堆料）。renwei 的检查清单（破折号/不是X而是Y/排比三连/万能展望）也已被 writing.md FORBIDDEN 全覆盖。

## 方法学 caveat（诚实记录）

补丁作为独立 `patch.md` 文件被 agent 读取并在 S1 交付物里按文件名引用，是测试设置的轻微 artifact（真实部署时该节内嵌 writing.md，不会被念名）。但核心失败（S3 内容级保守、S2 原创外溢）与命名无关，折掉 S1 的命名影响后 B 仍至少输 3 场，方向一致不噪，故 REJECT 稳健，无需重测。
