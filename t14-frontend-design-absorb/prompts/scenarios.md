# t14 场景定义

统一生成 prompt 模板（X = A 或 B）：

> 你是前端实现者。先完整阅读 `variant-X.md` 作为你必须遵守的设计规范，然后完成任务。产出单个自包含 HTML 文件（inline CSS，可含少量 vanilla JS）。完成后只返回：输出文件路径 + 开工时声明的 Design Read 与量化维度值。

## S1 设计感 landing（中文）
为独立开发者笔记工具「Driftnote」做中文 landing page。产品卖点：本地优先存储、Markdown 原生、双链笔记、一次买断 ¥99。目标用户：独立开发者和效率工具爱好者。要求有设计感，至少 5 个 section（hero / 功能 / 工作流演示 / 定价 / 收尾 CTA）。

## S2 portfolio（英文）
为一位拍城市建筑的摄影师 Mara Lindqvist 做英文个人作品集首页：hero + 精选作品（6 张，用 picsum 占位）+ about + 联系方式。要求让招聘她的画廊策展人记住这个站。

## S3 dashboard（反向场景，测增量是否产生噪音）
做一个服务器监控 dashboard 的 HTML 原型：顶部 4 个指标卡（CPU/内存/请求量/错误率）、中部时序图占位区、底部告警列表（含 loading/empty/error 状态示意）。内部运维工具，信息密度优先。

## 评分维度（judge 用）
1. 场景适配（拨盘/规则节选对没有，dashboard 是否克制）— 30%
2. 反 AI 审美（字体/配色/布局是否落入 Tell）— 30%
3. 设计完成度与记忆点（有没有一个真的被执行出来的差异化元素）— 25%
4. 可用性硬指标（对比度/focus/移动端/状态）— 15%

双盲：judge 看到的是「设计稿 1 / 设计稿 2」，A/B 顺序每场景随机。
