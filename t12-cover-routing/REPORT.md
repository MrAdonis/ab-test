## 关键发现

### 1. 「靠碰运气」前提在强模型 + 能力描述齐全时不成立
改动的论证依据是「三条散路没统一入口 → 帮我做一张封面走哪条完全靠碰」。AB 实测：条件 A（无这一行）下 Sonnet 对裸「帮我做一张封面」/「做个海报」**主动停下澄清**，没有「碰运气乱选」。能力描述（skill desc + memory 索引）已足够让模型正确推断，路由行没增加正确率。

### 2. B 的「三选一」措辞没有改变模型行为
Codex P0 担心「三选一」措辞暗示 dispatcher 应当直接挑一个，从而在裸请求上挑而不问。实测 B 仍然选择**澄清**而非硬挑——模型的「歧义即澄清」原生行为压过了路由行的措辞。即措辞没起负作用，也没起正作用。

### 3. 唯一可见的 B 微优势（未计入评分，因两侧都成功）
B 把三个目标路径 co-locate（`reference_x...md`+`/baoyu-infographic` / `/paper-layout` / `design-system §海报封面`）写在一格里；A 需从能力列表临时组装。两侧都成功，但 B 路径更确定、低方差——在 DEGRADING/CRITICAL context 档位或弱模型下是保险。

## 建议（留给用户拍板，未自动落地）

改动本身**保留**（无劣化 + 可发现性价值）。但因 AB 显示它没兑现核心承诺，**值得用 Codex 的 minimal fix 把它从「relabel」升级成「真决策程序」**，一次性也补 P2 coherence：

> 把「三选一」改为意图默认 + tiebreaker：
> 「**裸封面/海报/KV 默认 → ① AI生图成品**；明说要正式文档排版 → ② paper-layout；明说要给开发的参数规范 → ③ design-system。既要成品图又要实现参数 = 组合路由（①+③）。一页纸归 ② 不属封面类。」

代价≈一行措辞改动，把潜在风险（弱模型乱挑）和 P2 coherence 一并关掉。
