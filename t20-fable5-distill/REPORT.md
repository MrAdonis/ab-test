## 缘起

用户看到 X 推文（@wangray，2026-06-14）称"扒出 Fable 5 系统提示词，换上即可让 Opus 4.8 跑出 Fable 5 效果"。核查后判定核心说法不成立——系统提示词≠模型权重，那份文件是 claude.ai 消费端行为 prompt（product/refusal/tone/formatting + 大量产品工具定义），无法迁移 Mythos-class 能力，且原推方案 `--dangerously-skip-permissions` + 消费端 prompt 喂 Claude Code 是错配。改走"择取真正可迁移的工作风格条款并入现有 CLAUDE.md"路线，本测验证其中两条 prose 条款是否真增益。

## 设置

- **Variant A**: base 风格 persona（中文/简洁直给/有判断力编辑/绝对路径）
- **Variant B**: base + ①② 两条
- 唯一变量 = ①②；两版 base 完全相同，inline 进 prompt
- **生成**: 12 × Sonnet 干净上下文子代理（6 场景 × A/B），各写独立输出文件；显式禁读外部文件，防 CLAUDE.md 已含的 ①② 泄漏污染 A 版（此污染只会让 A/B 收敛、使测试偏保守，B 仍胜则可信）
- **评审**: 6 × Sonnet 盲评，输出复制为中性 `judge/sN-resp1/2.md`，每场景映射手动错开（resp1=：S1=B S2=A S3=B S4=A S5=B S6=A），judge 不可见变体身份，按场景定制四维权重

## 分析

**① 在"该散文化的多维内容"上是决定性差距（S1 +2.2）**：A 版把 event loop 讲成"有序列表+子列表+ASCII 流程图+代码块"四层骨架，judge 评"形式重于内容、像参考手册而非解释"（可读性 6.0 / 格式 5.5）；B 版散文带少量加粗，三概念自然衔接，还多补了"微任务嵌套微任务仍在本轮清空"的高价值细节（可读性 8.5 / 格式 8.5）。这就是 Fable"手感清爽"里 prompt 能贡献的那块。

**控制组 S3 是最关键验证：B 反而赢了（+1.6）**。本以为 anti-formatting 条款可能让 B 在"该用结构"的对比题上吃亏，结果相反——B 正确识别"三框架多维对比是结构所必需"，分段清晰逐框架对比（结构恰当 7.5）；A 反把三框架压成一段散文混排，judge 评"多维信息压成一行最难读，读者要自己拆"（结构恰当 4.5）。说明 ① 的"只有内容确实多维、列表是清晰所必需时才用结构"例外措辞站得住，B 没有盲目抹平结构。

**② 认错姿态两场全胜（S4 +0.8 / S5 +1.9）**：S5 最典型——B"我说错了，你查到的是对的。替代方案我可以帮你找，要继续吗？"（认错+主动承担+给决策点，9.0）；A"对，我说错了，建议查官方迁移文档"被 judge 评"把球踢给用户、实质推进最弱"（7.1）。② 的"留在问题上推进修复"直接起效。

**唯一败场 S2（−0.3）败因与条款无关**：两版格式分持平（9/9，均克制无过度），B 输纯因生成代理漏了 `ssh-agent`/`ssh-add` 步骤（macOS 实际必要），是生成波动不是 ① 导致。**唯一 watch-item**：judge 指出 B 的简洁"代价是漏了 ssh-agent"——① 的简洁压力理论上有"为短而牺牲正确性"的微弱风险，但本测中仅 −0.3 且简洁度本身被奖励（9.5 vs 8.5）。后续若发现 ① 让回复漏关键步骤，再给措辞补"简洁不牺牲正确/完整"。
