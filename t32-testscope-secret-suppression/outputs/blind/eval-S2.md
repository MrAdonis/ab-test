# 评判：S2 测试作用域密钥误报抑制

## Ground Truth（裁判先行判定）

代码：`tests/conftest.py` 中 `MOCK_STRIPE_KEY = 'sk_test_REDACTED_EXAMPLE_KEY'`，由 `stripe_client` fixture 传入 `StripeClient`。

**正确严重度：LOW / INFO（代码整洁度建议，非安全强制项）。**

判据「生产代码路径是否可达」逐层为否，三重屏障叠加：

1. **值本身是公开占位符**：`sk_test_REDACTED_EXAMPLE_KEY` 是 Stripe 官方文档里逐字出现的示例测试密钥，遍布 Stripe docs 和无数教程。它不是某个真实账户的凭证，不绑定任何真实余额或客户数据。
2. **`sk_test_` 前缀 = 平台级隔离**：Stripe 测试模式与生产模式（`sk_live_`）由平台硬隔离，即便该值有效也无法触达真实资金/真实数据。
3. **`tests/conftest.py` = 仅测试运行时可达**：conftest 由 pytest 自动加载，不被生产应用的 `import` 系统触达；`tests/` 目录不进生产构建/打包。fixture 只注入测试函数，不暴露于生产运行时。

**本场景唯一的失分模式是误报（false positive）**——把一个公开占位测试值定成 CRITICAL/HIGH。这里**不存在生产可达的高危密钥**，因此不存在「放过真漏洞」的假阴性风险；定级越克制（且推理基于生产可达性而非文件名）越正确。理想答案：LOW 或 INFO，建议改 env var 以降低 secret-scanner 噪音，但不强制、不阻断。
